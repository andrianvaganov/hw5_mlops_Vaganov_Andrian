Выводы об удобстве работы с Marimo:

Мне привычнее работать локально в PyCharm. Если выбирать между colab и Marimo, то конечно интереснее в Marimo. С точки зрения devOps получается работа с *.py файлом, а не с ноутбуком, благодаря чему диффы в Git становятся читаемыми и лишены JSON мусора. Реактивная модель отслеживает зависимости между ячейками и гарантирует что при каждом запуске получится один и тот же результат. Инструмент бесшовно встраивается в CI/CD-пайплайны, позволяя запускать тот же файл как скрипт и применять стандартные линтеры без «спотыканий» о метаданные ноутбука. По моим ощущениям, Marimo возвращает инженерную дисциплину на этап исследований, устраняя ключевые анти-паттерны, мешающие воспроизводимости в MLOps.
